# Environment Variables Setup

This project uses environment variables for email configuration. Follow the steps below to set up your local environment.

## Quick Setup

1. **Copy the example file:**
   ```bash
   cp .env.example .env.local
   ```

2. **Edit `.env.local`** with your actual credentials:
   ```env
   SENDER_EMAIL=your-sender@example.com
   SENDER_PASSWORD=your-password
   SENDER_SMTP_HOST=smtp-relay.sendinblue.com
   RECEIVER_EMAIL=your-receiver@example.com
   RECEIVER_PASSWORD=your-password
   ```

3. **Run tests:**
   ```bash
   npx playwright test
   ```

## Available Variables

| Variable | Description | Example |
|----------|-------------|---------|
| `SENDER_EMAIL` | Email address to send from | `sender@example.com` |
| `SENDER_PASSWORD` | App password for the sender account | `abc123...` |
| `SENDER_SMTP_HOST` | SMTP server host | `smtp-relay.sendinblue.com` |
| `RECEIVER_EMAIL` | Email address to receive test emails | `receiver@example.com` |
| `RECEIVER_PASSWORD` | IMAP password for receiving emails | `xyz789...` |

## Security Notes

- **NEVER commit `.env.local` or `.env`** to version control
- These files are in `.gitignore` - any changes will be safe from git
- Use GitHub Secrets for CI/CD pipelines
- Rotate credentials periodically
- Use application-specific passwords (not main account passwords)

## CI/CD Setup

For GitHub Actions, add secrets to your repository:

1. Go to **Settings → Secrets and variables → Actions**
2. Add these repository secrets:
   - `SENDER_EMAIL`
   - `SENDER_PASSWORD`
   - `SENDER_SMTP_HOST`
   - `RECEIVER_EMAIL`
   - `RECEIVER_PASSWORD`

## Email Provider Setup

### SendinBlue (Current Setup)

1. Sign up at [SendinBlue](https://www.sendinblue.com/)
2. Go to **Parameters & SMTP** → **SMTP**
3. Generate your API key
4. Use `smtp-relay.sendinblue.com` as the host

### Gmail (For Receiving)

1. Enable **2-Step Verification**
2. Generate an **App Password**
3. Use this app password in `RECEIVER_PASSWORD`
4. Use `imap.gmail.com:993` for IMAP host

## Running Email Integration Tests

The package includes integration tests that actually send and receive emails:

```bash
# Run all tests (skips email tests if credentials not configured)
npx playwright test

# Run only email integration tests
npx playwright test tests/email-integration.spec.ts

# Run with verbose output
npx playwright test tests/email-integration.spec.ts --project=chromium
```

**Note:** Email integration tests will automatically skip if credentials are not configured in `.env` file. The filter logic tests in `email-filters.spec.ts` are unit tests and don't require credentials.
